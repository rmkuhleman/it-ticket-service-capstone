# IT Ticket Service

The purpose of this project is to create a web-based service that provides users with an easy and structured way to request help with IT and technical issues. The system allows users to submit tickets, track their status, and provide feedback once their issue has been resolved. 

This service is designed to be simple, scalable, and adaptable for use in small organizations, with the potential to expand to larger environments such as businesses or healthcare settings.


## Main Features

- Secure user authentication (username and password)
- Ticket submission form with validation
- Issue categories and subcategories
- User-specific ticket tracking
- Admin/technician dashboard for managing all tickets
- Ticket status workflow (Open → In Progress → Escalated → Closed)
- Escalation handling for unresolved issues
- Resolution notes for completed tickets
- User feedback system for closed tickets
- Filtering and search functionality for tickets


## Non-Features

- Real company or enterprise system integration
- Advanced account management (password reset, email verification)
- AI chatbot or automation features
- Email notifications or alerts
- Mobile application version
- Complex role hierarchies beyond user/admin


## Success Criteria

A user should be able to securely log in, submit a ticket, and view only their own tickets. A technician (admin) should be able to review, update, escalate, and resolve tickets through the admin dashboard. Once a ticket is marked as closed, the user should be able to provide feedback on the service.

The system should demonstrate a complete ticket lifecycle, proper access control, and consistent functionality across all user and admin workflows.
